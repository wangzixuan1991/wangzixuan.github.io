LOAD BALANCING REDESIGN

A guided expereince for load balancing configuration.

Problem

There are 2 different options to distribute network traffic using Microsoft Azure - Load Balancer and Application Gateway. These options work differently from each other, having a different feature set and support different scenarios. They can each be used in isolation, or combining them. But our users don't know which to choose. Even if they picked the right thing for their scenario, the success rate of configuring a load balancer/application gateway is lower than 50% according to usability studies. Fun fact: we once did a workshop with PMs from compute and network API team, almost all of them failed the task to set up a load balancer.

Current load balancer setup expereience

Problem with current design:

  • Load balancer and application gateway have totally separated entry points in Azure. We don't provide an experience for them to compare and choose.
  • Users only specifies the minimum information when creating a load balancer, and there's no guided experience for configuration after they finish creating it.

Conceptual sketches

With PM and devs, we did a quick concept brainstorming and pulled together some concetptual sketches.

Task flow

I created a task flow taking in consideration of two main scenarios:

  • New users who needs a recommendation of what type to use. They would want to finish create quickly and configure later.
  • Frequent users who know what they need and want to set up everything at once.
